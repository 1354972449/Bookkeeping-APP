# 💰 黑马记账 - 个人财务管家

> 一款简洁、专业、全中文的桌面个人记账应用，帮助您轻松记录每一笔消费，养成理财好习惯。

---

## ✨ 功能特性

### 📒 记账功能
- **极速记账**：两步完成记账（输入金额 → 选择分类），支持快捷金额一键选择
- **📅 任意日期补记**：支持选择历史任意日期（某月某日）进行补记账单，不再局限于当日记账
- **➕ 自定义分类**：可随时新增自定义主分类（选图标）和子分类，满足个性化记账需求
- **🏷️ 丰富分类**：内置 **15 大主分类 + 数十个子分类**，覆盖餐饮、交通、购物、居住、人情、育儿、宠物等生活方方面面
- **📝 账单备注**：每笔支出支持添加文字备注，方便日后复盘消费场景

### 📊 多维度数据统计
- **🔄 三种统计维度**：支持 **按年 / 按月 / 按天** 切换查看消费数据
- **📈 趋势图表**：年度各月支出趋势线图、月度每日支出趋势、当日分类对比柱状图
- **🥧 可视化报表**：饼图 + 柱状图直观展示消费分布，自动识别冠军分类
- **💡 智能理财建议**：根据年/月/日不同维度的支出金额，自动生成个性化消费建议
- **🏆 分类排行榜**：按消费金额排序展示分类排名，Top1 重点高亮提示
- **📋 账单管理**：按日期分组展示全部账单，支持删除记录，「今日账单」专区一目了然
- **💰 实时概览**：首页顶部卡片实时显示「今日支出 / 本月支出 / 记账笔数」三大核心指标

### 🎮 工作之余 · 放松专区
- **🐍 贪吃蛇小游戏**：内置经典贪吃蛇，记账间隙放松大脑
  - 4 档难度可选：简单 / 普通 / 困难 / 地狱
  - 键盘方向键 / WASD / 触屏按钮 三种操控方式
  - 自动保存历史最高分（localStorage 本地持久化）
  - 空格键一键暂停 / 继续 / 重开

### 🛡️ 隐私与稳定性
- **🖥️ 全中文界面**：Electron 菜单栏、按钮文案、提示信息全部中文化，无任何英文残留
- **💽 纯本地存储**：基于 SQLite（sql.js）纯本地数据库，隐私数据 100% 留在您的电脑上
- **🎨 精美 UI**：紫蓝渐变主题色、圆角卡片阴影、脉冲悬浮按钮、丝滑过渡动画
- **🛡️ 6 层兜底保护**：数据库损坏 / 文件被占用 / 分类丢失等极端场景自动修复，永不空白

## 🚀 快速开始

### 环境要求

- Node.js **≥ 18**（推荐 20+）
- npm 或 pnpm / yarn
- Windows 10 / 11 64 位

### 1. 安装依赖

```bash
cd 黑马记账App
npm install
```

### 2. 三种方式 · 直接启动 APP 🚀

#### ✅ 方式一：免安装绿色版（最快、最省事）
**无需 Node 环境、无需编译、无需安装！** 直接双击即可运行：

```
release/win-unpacked/黑马记账.exe
```

> 💡 方便分享：把整个 `win-unpacked` 文件夹压缩发给朋友，解压后双击 exe 就能用。

#### ✅ 方式二：标准安装包（推荐分发）
运行安装包，按提示完成安装：

```
release/黑马记账 Setup 1.0.0.exe
```

安装完成后：
- 桌面会出现「黑马记账」快捷方式（带自定义紫蓝钱包图标）
- 开始菜单 → 「黑马记账」文件夹 → 启动
- Windows 任务栏会正确显示自定义 APP 图标（已配置 AppUserModelID）

#### ✅ 方式三：开发模式运行（改代码调试时用）

```bash
npm run dev
```

> 此命令会同时启动两件事：
> - Webpack Dev Server（热更新，端口 3000）
> - Electron 桌面窗口（等待前端服务就绪后自动弹出）
>
> 修改代码保存后会自动热重载，无需重启。

### 3. 打包成 Windows 可执行文件

```bash
# 打包前端代码 + 生成 Windows 安装包
npm run build:win
```

打包完成后，产物位于项目根目录的 `release/` 文件夹：

| 文件/目录 | 说明 |
|---|---|
| `黑马记账 Setup 1.0.0.exe` | **推荐**：NSIS 图形化安装向导，支持自定义安装目录，自动创建桌面/开始菜单快捷方式，图标为自定义紫蓝钱包设计 |
| `win-unpacked/黑马记账.exe` | 绿色免安装版，直接双击运行，不用安装 |

### 4. 仅构建前端代码（不打包 Electron）

```bash
npm run react:build
```

产物输出到 `dist/` 目录。

## 📖 使用指南

### 🎯 如何新增自定义分类？
1. 点击底部「记账」进入记账弹窗
2. 在选择分类页面点击右上角的 **➕ 新增大类** 或 **➕ 新增子分类**
3. 输入分类名称（大类还可选择个性图标），点击「确认添加」即可
4. 新增的分类会立即出现在分类列表中，可随时用于记账

> 💡 提示：自定义分类在数据库结构重建时会被自动保留，不会丢失。

### 🎯 如何补记历史账单？
1. 点击「记账」打开弹窗后，顶部会出现「记账日期」选择器
2. 点击日期框即可弹出日历，选择任意历史日期（不超过今天）
3. 正常输入金额、选择分类后确认，账单即计入所选日期

### 🎯 如何查看按年 / 按天统计？
1. 切换到底部「统计」Tab
2. 页面顶部有 **按年 | 按月 | 按天** 的分段切换器
3. 点击切换即可查看对应维度：
   - **按年**：年度各月支出趋势 + 年度分类分布 + 月均消费
   - **按月**：月度每日支出趋势 + 月度分类分布 + 日均消费
   - **按天**：当日分类柱状对比 + 单笔数统计
4. 左侧和右侧箭头按钮可前后切换年份 / 月份 / 日期

### 🎯 如何玩贪吃蛇？
1. 点击底部「放松」Tab 进入游戏
2. 选择难度（简单 / 普通 / 困难 / 地狱），速度越快得分挑战越大
3. 点击「开始游戏」或按空格键开始
4. 操控方式：
   - 键盘：↑↓←→ 方向键 或 W A S D
   - 鼠标/触屏：点击右侧面板的 ▲▼◀▶ 方向按钮
   - 空格键：暂停 / 继续 / 重开
5. 目标：吃红色食物变长，每吃一个 +10 分，不要撞墙或撞到自己！

## ❓ 常见问题

### 1. 启动后分类空白 / 选择不了分类

**原因：** 旧版本数据库结构损坏或分类表不兼容。
**解决：** 关闭 APP → 删除 `heimazhangji.db` 文件 → 重新启动 APP，会自动重建 15 大完整分类。

### 2. 新增的自定义分类会丢失吗？

不会。自定义分类带有 `is_custom = 1` 标记，每次数据库结构重建时会单独备份自定义分类并恢复，不会影响用户自定义数据。

### 3. 端口 3000 被占用，`npm run dev` 启动失败

解决方法二选一：
- 关闭占用 3000 端口的程序（任务管理器 → 详细信息 → 结束 node.exe）
- 或修改 `webpack.config.js` 中的 `devServer.port` 换一个端口（同时注意 `package.json` 里 `dev:electron` 脚本中的 `wait-on http://localhost:3000` 也要同步修改）

### 4. `npm` 命令找不到

请先安装 Node.js（LTS 版，官网下载），安装完成后**重启终端**再试。

### 5. 安装包 / 打包时报图标错误

**常见错误：** `Icon must be at least 256x256 pixels` 或 `icon is not a valid PNG`

**解决方案：**
- 确认 `public/icon.png` 文件存在，并且**文件头是真正的 PNG 格式**（`89 50 4E 47`，不能直接改后缀名伪装）
- 推荐尺寸：**512x512 像素**，electron-builder 会自动生成 256/128/64/48/32/16 多尺寸 ICO
- 如果图标生成有问题，可以使用 C# / System.Drawing 自己绘制 PNG，确保格式正确

### 6. 安装后桌面快捷方式 / 任务栏图标不显示自定义图标

**典型症状：** 运行 `黑马记账.exe` 后，任务栏显示的是 Electron 默认蓝色图标，而非自定义的紫蓝钱包图标。

**根因：** Windows 任务栏的图标缓存问题，或缺少 AppUserModelID 绑定。

**解决方案：**
1. **代码已内置修复**（v1.1.1 起）：主进程启动时已调用 `app.setAppUserModelId('com.heima.zhangji.1.0.0')`，与 package.json 的 `appId` 对应
2. 手动刷新 Windows 图标缓存：
   - 按 `Win + R` → 输入 `ie4uinit.exe -show` 回车（立即刷新图标缓存）
   - 或右键快捷方式 → 属性 → 更换图标 → 重新确认
3. 若之前安装过旧版本，请先**卸载旧版**再安装新版，避免缓存复用
4. 也可直接使用 `release/win-unpacked/黑马记账.exe` 绿色版，图标由 exe 资源直接加载，无需等待系统缓存

### 7. 打包后打开提示数据库 / WASM 加载失败

已在 `package.json` 的 `build.asarUnpack` 和 `extraResources` 中配置了 `sql-wasm.wasm` 的释放路径，请使用 `npm run build:win` 重新打包；开发模式下直接读取 `node_modules/sql.js/dist/sql-wasm.wasm`。

### 8. 贪吃蛇最高分会清除吗？

最高分通过 `localStorage` 保存在浏览器内核的本地存储中，只要不卸载 APP 或手动清空缓存，最高分将一直保留。

## 🛡️ 6 层分类兜底机制

为彻底解决「分类丢失无法记账」的问题，代码中内置全链路兜底：

| 层级 | 位置 | 作用 |
|---|---|---|
| 第 1 层 | `database.js` → `initDatabase` | 打开 .db 文件失败自动备份 + 新建空库；建表/初始化分类失败自动重建 |
| 第 2 层 | `database.js` → `saveDatabase` | 写盘 EPERM 被占用 → 写 tmp 文件再 rename 替换，绝不写空 |
| 第 3 层 | `database.js` → `getAllCategories` | db 未初始化 / 查询异常 / 大类为空 / 子类全空 → 返回内置 15 大类静态数据 + 尝试重写数据库 |
| 第 4 层 | `database.js` → `addRecord` | 写记录前校验：分类表空 OR 分类 ID 不存在 → 立即重建分类表再写入（重建时保留用户自定义分类） |
| 第 5 层 | `renderer/utils/database.ts` | electronAPI 不存在 / IPC 抛错 / 返回空 / 无 children → 前端内置 15 大类兜底显示 |
| 第 6 层 | 全代码 SQL 访问 | 所有 `result?.[0]?.values?.[0]?.[0]` 加可选链，防止 undefined 导致白屏 |
